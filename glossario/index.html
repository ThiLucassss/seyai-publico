Glossário Técnico SeyAI

Cursor + Terminal + Git para sócios e novos integrantes do projeto. v1.0 · 2026-04-20

0 · Como usar este guia

Leia em ordem se está começando do zero. Use como referência (Ctrl+F) depois que dominar o básico.

Cada seção tem nível de dificuldade: básico usa todo dia · médio usa toda semana · avançado cuidado redobrado.

2 · Cursor — primeiro contato

Cursor é o editor que você usa agora. Baseado no VS Code, com IA nativa. Roda Claude Code dentro do terminal integrado.

2.1 Atalhos do dia a dia básico

No Mac: troque Ctrl por Cmd.

Navegar

  • Ctrl + P — abrir arquivo por nome
  • Ctrl + Shift + P — paleta de comandos (faz qualquer coisa do Cursor)
  • Ctrl + B — mostrar/esconder barra lateral
  • Ctrl + J — mostrar/esconder terminal
  • Ctrl + Tab — alternar entre abas abertas

Editar

  • Ctrl + S — salvar
  • Ctrl + D — selecionar próxima ocorrência da palavra (multi-cursor)
  • Alt + ↑/↓ — mover linha
  • Shift + Alt + ↑/↓ — duplicar linha
  • Ctrl + / — comentar/descomentar
  • Ctrl + Z / Ctrl + Y — desfazer / refazer

Buscar

  • Ctrl + F — buscar no arquivo atual
  • Ctrl + H — buscar e substituir
  • Ctrl + Shift + F — buscar em todo o projeto

2.2 Trabalhar com IA dentro do Cursor básico

  • Ctrl + L — abre chat lateral com IA
  • Ctrl + K — IA inline (edita código onde o cursor está)
  • Ctrl + I — Composer (IA edita múltiplos arquivos)
  • Tab — aceitar sugestão de autocomplete
  • Esc — rejeitar sugestão
  • @ no chat — mencionar arquivo, pasta ou código específico
exemplo
Selecione um trecho → Ctrl + K → "transforma em função" → IA edita ali mesmo.

2.3 Terminal integrado médio

É onde você roda Claude Code e comandos de git.

  • Ctrl + ` — abrir novo terminal
  • Ícone de split — dividir terminal lado a lado
  • Ícone de lixeira — fechar terminal
  • cls (Win) / clear (Mac) — limpar tela

Claude Code dentro do Cursor

  • claude — iniciar sessão
  • /login — autenticar
  • /effort low|medium|high — ajustar esforço de raciocínio
  • /fast — modo rápido (Opus 4.6)
  • /clear — limpar contexto da conversa
  • /help — ver ajuda
dica
Digite / no Claude Code pra ver os 21 comandos do SeyAI (/thiago, /lima, /shion, /atualizacao, /push, /puxar-bautitz etc).

2.4 Configurações importantes avançado

  • Privacy Mode ON (recomendado pro SeyAI): Cursor Settings → Privacy → Privacy Mode: Enabled. Garante que código não é usado pra treinar modelos.
  • .cursor/rules/ — regras de IA específicas do projeto.
  • .vscode/settings.json — configurações do projeto (Cursor é compatível).

4 · Git — versionar trabalho

4.1 Setup inicial (uma vez na vida) básico

git config --global user.name "Thiago Freitas"
git config --global user.email "freitas4thiagotlg@gmail.com"

4.2 Ciclo do dia a dia básico

receita
status → add → commit → push. Memorize.

Trazer atualizações

  • git clone <url> — cópia inicial completa (uma vez por projeto)
  • git fetch — baixa novidades sem aplicar
  • git pull — baixa e aplica na sua branch

Salvar trabalho

  • git status — vê o estado atual (use SEMPRE antes/depois)
  • git diff — vê o que mudou linha por linha
  • git add arquivo — marca arquivo pro próximo commit
  • git add . — marca tudo da pasta atual
  • git commit -m "msg" — grava snapshot local

Enviar pro GitHub

  • git push — manda commits locais pro remoto
  • git push -u origin thiago — manda E vincula sua branch local à remota (1ª vez)
fluxo padrão
git status
git add _seyai_output/thiago/inbox/proposta.md
git commit -m "feat: PROP-THIAGO-005 nova proposta"
git push

4.3 Branches médio

Branch = linha paralela de trabalho. Você está na thiago; a canônica é Bautitz.

  • git branch — lista branches locais
  • git branch -a — lista locais + remotas
  • git switch nome — troca pra branch existente
  • git switch -c nova — cria branch nova e troca
  • git checkout -b nova — mesmo que acima (forma antiga, ainda comum)
  • git branch -d nome — apaga branch (segura)
exemplo real
git checkout -b thiago        # cria branch "thiago"
git push -u origin thiago     # publica no GitHub e vincula

4.4 Combinar trabalho médio

  • git merge Bautitz — traz commits da Bautitz pra branch atual
  • git rebase Bautitz — reescreve histórico "por cima" da Bautitz (linear)
  • git stash — guarda mudanças em gaveta temporária
  • git stash pop — recupera o que tava na gaveta

4.5 Inspecionar histórico médio

  • git log --oneline -10 — últimos 10 commits resumidos
  • git log --graph --oneline — histórico visual com branches
  • git diff Bautitz..thiago — diferenças entre branches
  • git blame arquivo — quem mudou cada linha

6 · Nunca faça isso

Lista única de armadilhas. Lê uma vez, nunca esquece.

ComandoPor que evitar
rm -rf / rmdir /sNão vai pra lixeira. Some pra sempre.
git push --force em Bautitz ou mainSobrescreve trabalho de Marcos e dos outros.
git reset --hard HEAD~1 sem certezaApaga commits + mudanças locais.
git checkout .Descarta TODAS mudanças não commitadas. Sem volta.
git clean -fdApaga arquivos não rastreados. Sem lixeira.
git branch -D nomeApaga branch forçado, perde commits não mergeados.
git add -A em repo grandePode incluir lixo (.env, builds, segredos).
Commit com .env ou credenciaisVaza segredos pro GitHub. Praticamente impossível remover do histórico.
Editar arquivo de outro sócio em _seyai_output/lima/Viola UNI-056 / UNI-059 / UNI-060.
versão segura de force push
git push --force-with-lease — aborta se alguém commitou depois de você.

9 · Glossário alfabético

Pra consulta rápida. Termos em ordem A-Z.

add
Comando git que marca arquivos pro próximo commit (staging).
branch
Linha paralela de trabalho no git.
CLI
Command Line Interface, programa que roda no terminal.
clone
Cópia inicial completa de um repositório remoto.
commit
Snapshot gravado no histórico local.
conflict
Git não conseguiu juntar mudanças automaticamente.
diff
Diferença entre duas versões.
fetch
Baixa novidades do remoto sem aplicar.
fork
Cópia independente de um repo no GitHub.
gitignore
Arquivo que lista o que git deve ignorar.
hash / SHA
Identificador único do commit (ex: 6dacc3c).
HEAD
Ponteiro pra "onde você está" no histórico.
merge
Junta uma branch dentro da outra.
origin
Apelido padrão do remoto principal.
PR (Pull Request)
Pedido pra mergear branch na principal.
pull
fetch + merge na sua branch atual.
push
Envia commits locais pro remoto.
rebase
Reescreve histórico "por cima" de outra branch.
reflog
Log interno do git, salva-vidas pra recuperar coisas perdidas.
remote
Servidor remoto (GitHub).
repo
Repositório, projeto versionado.
stash
Gaveta temporária pra mudanças não commitadas.
staging / index
Cesta de arquivos prontos pra commit.
switch
Troca de branch (forma moderna do checkout).
tag
Apelido fixo num commit (ex: v0.9.0).
terminal
Janela onde você digita comandos.
upstream
Remoto "pai" quando você fez fork.
working tree
Seus arquivos como estão agora no disco.
